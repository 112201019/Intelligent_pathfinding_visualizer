Maze Visualizer Help

Explore pathfinding algorithms visually! Generate a maze and watch different algorithms find a route from the start (violet) to the end (blue).

Getting Started

  • Click to generate a random maze.
  • Choose a pathfinding algorithm from the dropdown menu.
  • Click to begin the visualization.
  • Use / to control the animation.
  • Click to view performance metrics.

Maze Interaction

  • Shift + Click a cell to set the Start position.
  • Ctrl + Click (or Cmd + Click on Mac) a cell to set the End position.
  • Clicking a cell normally currently has no action.

Cell Colors

  • Start position
  • End position
  • Visited cells during search
  • Final Path cells
  • Walls

Pathfinding Algorithms

Depth-First Search (DFS): Explores as far as possible along each branch before backtracking. Very fast, often finds *a* path quickly, but rarely the shortest one.

Breadth-First Search (BFS): Explores level by level, checking all neighbors before moving deeper. Guarantees the shortest path (in terms of number of steps) for unweighted mazes.

A* Search: A smarter search algorithm that uses a heuristic (an estimated cost to the goal) to prioritize exploring cells that seem closer to the end. Guarantees the shortest path if the heuristic is admissible (never overestimates the cost).

  • Manhattan: Estimates cost as `dx + dy`. Good for grids where only horizontal/vertical movement is allowed.
  • Euclidean: Straight-line distance `sqrt(dx¬≤ + dy¬≤)`. More accurate distance but computationally slightly heavier.
  • Chebyshev: Estimates cost as `max(dx, dy)`. Useful when diagonal movement has the same cost as straight.
  • ManWallHeu: A custom heuristic (Manhattan + Wall Penalty) that might perform differently depending on maze structure (implementation details vary).
